System Events

Event: Frame Received

Triggered when cam.get_latest_frame() returns a valid frame. The timestamp is checked against the last processed timestamp to ensure we don't re-process old data.

Event: Frame Processed

The vision pipeline activates. Landmarks are extracted, and the looking boolean is updated. This is the most computationally intensive phase.

Event: No Frame Timeout

Safety mechanism: If time.monotonic() - last_seen > 1.0, the system assumes the camera has frozen or disconnected and initiates a safe shutdown.
